Code Analysis 2021.r1 - June 2021 (4Dv18)

General

  • Converted to 4D v18.
  • Major re-write moving away from process arrays to objects.
  • Removed all previous references to the internal arrays that were used.
  • Reworked the UI on the two main windows.
  • Recongnize /* ... */ as comments.

Code Analysis Window

  • Analysis tab removed since that information is available on the Explorer Window.
  • When exporting the methods as text, the structure is also exported.
  • Added a button to diff save the results on the Difference tab.

Explorer Window

  • Reworked the UI by using tabs on the explorer window:
    • "Methods" tab - provides a quick way to find methods as well as see some basic static analysis statistics on each one
    • "Structure" tab - provides a list of all the tables, fields, types of indexes as well as the abiltiy to get the ratio of unique values
    • "Summary" tab - provided statistics from the host structure
  • User can right click a method on the Methods tab to view the methods that call the method. Selecting the caller will open in the code editor.

Miscellaneous

  • Added componentManifest.json file.
    See https://github.com/dbeaubien/Manifest-Generator-4D-Component for more details.

Preferences

  • Reworked the layout of across all the tabs. Added a new tab called "General".
  • Added preferences on the General tab to configure what happens when the "CA_OnStartup" method is called.

Components Methods Shared to the Host

  • Added "CA_OnStartup". This function is meant to be added to the host database's "On Startup" method. It will perform the startup actions defined in the preferences.
  • Added "CA_SaveStructureDetails". This function exports the structure to the optionally specified folder. If no folder is specifed then the structure is exports to the folder defined in the preferences.
  • Added "Structure_GetFieldIndexRatio". This function returns the uniqueness ratio for the specified field. The returned ratio is range between 1.00 (100% unique) to 0 (1 value for every record).

Code Analysis v1.6 - Mar 11, 2015

Method Analysis Engine

  • Completely refactored the code that does all the analysis of the methods reducing complexity and removing duplication.
  • The results of the analysis is now persisted to the local disk along with the other Code Analysis component preferences.
  • Refreshing the analysis only performs updates on the methods that have been updates since the last time the analysis was run. This is determined by using the modification data and time as reported by 4D.
  • Exposed "MethodStats_RecalculateAll" to the host database. Calling this method does a full recalculation of the method statistics in the host database.
  • Exposed "MethodStats_RecalculateModified" to the host database. Calling this method does a recalculation of the method statistics of the methods in the host database that have been modified since the last recalculation.
  • Improved detection algorithm used to determine if the required project form has been copied to the host database prior to exporting the properties. The message displayed to the user is improved as well.

Complexity

  • Added a "View Graphs" button that is enabled once the code has been analysed. Pressing this button will open a dialog that will allow you to view a graph of the # of methods by lines of code and # of methods by complexity. These graphs can be used to have a higher level view of some metrics of your code.
  • The results of the last analysis is persisted and restored when the main Code Analysis window is opened.
  • Holding down the option or alt key and pressing the "Analyse Code" button will bring up a prompt that will allow you to select if only the update methods or all the methods will be analysed.

Method DIFF

  • Fixed bug where a method diff window would not open if the method name matched an already running process. The process name is prepended with "diff:".

Method Export

  • Altered how methods are exported. The component will not do a full replace of the contents of the export folder. Only the method files that differ will be exported. The change has been made to improve the workflow when using external code repositories.
  • Added a check to ensure that the folder the methods are being exported to is writable. An error is displayed if it is not.

Structure Export

  • Added a "Index Field Stats". Using this button will perform a code scan of they entire code base and will save two reports to the default export folder. One report contains a full listing of the fields that are used in query statements along with a count and if the field is indexed or not. The 2nd report shows the methods (and line) that each field query is performed in.
  • Fixed bug that caused no index information to be exported in the table/field info export file.

Methods That Are Shared With Host Database

  • "Structure_SaveStructDefn2Folder" is now shared with the host database. This method will export the structure definition to a file. See the Component API Docs for more information.
  • "Structure_ScanIndexedFieldUsage" is a new method that is shared with the host database. This method will scan the host database for where fields are used in queries and produce a report that shows usage and if they are indexed or not. See the Component API Docs for more information.
  • "MethodStats_RecalculateAll" is a new method that is shared with the host database. This method will recalculate all the method metrics that the component captures. The results are stored back to disk for future usage. See the Component API Docs for more information.
  • "MethodStats_RecalculateModified" is a new method that is shared with the host database. This method will loads the previously method metrics and updates the metrics on the methods that have been updated since the last time the calculations where performed. See the Component API Docs for more information.

Code Analysis v1.0 - Oct 5, 2012

Code Analysis

  • Allow developer to gather statistics (# of code lines, # of comment lines, # of blank lines, Cyclomatic Complexity) of every project, form, object, trigger and database method in the structure.
  • Allow developer to save gathered statistics to a csv file.
  • Allow developer to open methods directly from the list of methods by double clicking or hitting the return key.
  • Navigation keys are supported to navigate the list of methods (e.g up/down arrow, page up/down).
  • Allow developer to set the thresholds for the Cyclomatic Complexity via preferences.
  • Allow developer to set to include the Compiler methods in the analysis via preferences.

DIFF Analysis

  • Allow developer to compare the methods in the current structure against code files that have been saved to disk. Methods are matched based on file name with an extension of ".txt".
  • Allow developer to not report on methods that are only in the structure and not on the disk.
  • Allow developer to not report on methods that are on the disk and not in the structure.
  • Allow developer to exclude the "attribute" line when looking for changes against the files stored on disk.
  • Allow developer to examine each method's differences from the file stored on disk. There are options to ignore blank lines, comment lines and the attribute line.
  • Allow developer to open the method in the structure and/or open the stored method on disk when examining the difference details.
  • Allow developer to set a preference for the # of spaces to use for indentation when viewing the details of the method differences.

Exporting Code

  • Allow developer to export all the methods of every project, form, object, trigger and database method in the structure to disk.
  • Allow developer to export HTML documentation of the methods of every project, form, object, trigger and database method in the structure to disk.
  • Allow developer to export HTML documentation of the project methods that have been marked as "shared by components and host database" to disk.
  • Allow developer to specify a preference for the root folder where all exported files are to be placed.
  • Allow developer to specify a preference for the folder (within the root folder) that the exported methods will be contained.
